# Architecture

## High-Level Pattern

This is a content-first static site architecture built around Astro routes, content collections, and reusable presentation components. Most logic runs at build time; client-side JavaScript is used selectively for theme switching, search UI, comments, and a few interactive enhancements.

## Main Layers

## 1. Configuration Layer

- [`astro.config.ts`](../../astro.config.ts) defines integrations, markdown plugins, and Vite behavior.
- [`src/site.config.ts`](../../src/site.config.ts) centralizes site metadata, nav links, comment toggles, and expressive-code theme settings.
- [`src/content.config.ts`](../../src/content.config.ts) defines collection schemas and frontmatter validation.

## 2. Content Layer

- Source content lives under:
  - [`src/content/blog`](../../src/content/blog)
  - [`src/content/note`](../../src/content/note)
  - [`src/content/tag`](../../src/content/tag)
- Content is loaded with Astro collections and transformed by remark/rehype plugins during build.

## 3. Data Derivation Layer

- [`src/data/post.ts`](../../src/data/post.ts) is the main content query module.
- It provides:
  - `getAllPosts()`
  - `getBacklinksForPost()`
  - tag aggregation helpers
  - yearly grouping helpers
- Backlinks are derived by rendering posts and reading custom frontmatter generated by `remarkPostBacklinks`.

## 4. Route Layer

Entry points are Astro pages under `src/pages/`:

- [`src/pages/index.astro`](../../src/pages/index.astro): homepage
- [`src/pages/about.astro`](../../src/pages/about.astro): static about page
- [`src/pages/projects/index.astro`](../../src/pages/projects/index.astro): project showcase
- [`src/pages/blog/[...page].astro`](../../src/pages/blog/[...page].astro): blog archive
- [`src/pages/blog/[...slug].astro`](../../src/pages/blog/[...slug].astro): individual blog posts
- [`src/pages/notes/[...page].astro`](../../src/pages/notes/[...page].astro): notes archive
- [`src/pages/notes/[...slug].astro`](../../src/pages/notes/[...slug].astro): individual notes
- [`src/pages/tags/index.astro`](../../src/pages/tags/index.astro): tag listing
- [`src/pages/tags/[tag]/[...page].astro`](../../src/pages/tags/[tag]/[...page].astro): per-tag archives
- [`src/pages/rss.xml.ts`](../../src/pages/rss.xml.ts) and [`src/pages/notes/rss.xml.ts`](../../src/pages/notes/rss.xml.ts): feeds
- [`src/pages/og-image/[...slug].png.ts`](../../src/pages/og-image/[...slug].png.ts): generated social images

## 5. Layout And Component Layer

- [`src/layouts/Base.astro`](../../src/layouts/Base.astro) wraps all pages with head/meta, header, footer, skip link, speculation rules, and pangu spacing.
- [`src/layouts/BlogPost.astro`](../../src/layouts/BlogPost.astro) adds post-specific UI such as masthead, TOC, TLDR, backlinks, comments, and back-to-top behavior.
- Components are grouped by concern:
  - `src/components/blog/`
  - `src/components/layout/`
  - `src/components/home/`
  - `src/components/projects/`
  - `src/components/note/`

## Data Flow

For a blog post request:

1. `getStaticPaths()` in [`src/pages/blog/[...slug].astro`](../../src/pages/blog/[...slug].astro) enumerates posts from `getAllPosts()`.
2. Astro `render(post)` converts MD/MDX content to `Content`, headings, and remark frontmatter.
3. [`src/layouts/BlogPost.astro`](../../src/layouts/BlogPost.astro) composes rendered content with metadata, TLDR, TOC, backlinks, and optional Giscus.
4. [`src/components/BaseHead.astro`](../../src/components/BaseHead.astro) emits canonical, Open Graph, Twitter, feed, and manifest metadata.

## Plugin-Driven Augmentation

Custom plugins extend raw content before it reaches layouts:

- [`src/plugins/remark-reading-time.ts`](../../src/plugins/remark-reading-time.ts) injects reading-time text.
- [`src/plugins/remark-post-backlinks.ts`](../../src/plugins/remark-post-backlinks.ts) tracks outbound internal blog links.
- [`src/plugins/remark-admonitions.ts`](../../src/plugins/remark-admonitions.ts) converts directives into semantic admonition HTML.
- [`src/plugins/remark-github-card.ts`](../../src/plugins/remark-github-card.ts) transforms directives into GitHub card markup plus client-side fetch logic.

## Client-Side Islands / Enhancement Model

This repo uses Astro’s “mostly static, selectively interactive” model:

- Theme initialization and persistence: [`src/components/ThemeProvider.astro`](../../src/components/ThemeProvider.astro)
- Theme toggle web component: [`src/components/ThemeToggle.astro`](../../src/components/ThemeToggle.astro)
- Search dialog custom element: [`src/components/Search.astro`](../../src/components/Search.astro)
- Giscus embed loader: [`src/components/blog/Giscus.astro`](../../src/components/blog/Giscus.astro)

There is no centralized client state store; each enhancement is isolated and DOM-driven.
